# Contributing

Thank you for considering contributing to this project! We welcome all kinds of contributions.

## Getting Started

1. **Fork the repository** and create your branch:
   `git checkout -b feature/your-feature-name`

2. **Install dependencies** listed in `requirements/` and run tests before submitting.

3. **Follow code style guidelines**:
   Keep your code clean and consistent with the existing style.

## Commit Messages

* Use clear, descriptive commit messages.
* Follow this format:
  `type: short summary`
  Example: `fix: handle timeout error in async_task`

Common types:

* `feat`: new feature
* `fix`: bug fix
* `docs`: documentation changes
* `refactor`: code restructuring (no logical change)
* `test`: adding or updating tests
* `chore`: general maintenance

## Pull Requests

* Keep PRs focused and minimal.
* Reference related issues when applicable.
* Ensure your branch is up-to-date with `main`.
